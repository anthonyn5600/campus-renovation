# CSUF Campus Navigator v2

Interactive campus map, event discovery, food directory, and contact directory for Cal State Fullerton. Complete ground-up rebuild of the [original CampusRenovation](https://github.com/anthonyn5600/CampusRenovation) project.

## What Changed

| | v1 (Original) | v2 (This Repo) |
|---|---|---|
| **Stack** | Vanilla HTML/CSS/JS, no build system | Next.js 16 + TypeScript + Tailwind CSS |
| **Map** | Leaflet with 360 lines of manual marker code | react-leaflet with data-driven markers (~30 lines) |
| **Routing** | Mapbox API with client-side token | Simple polyline + haversine distance calc, zero API keys |
| **Events** | Selenium scraping → flat file → manual Gist upload | Direct fetch from CSUF's public JSON API, build-time download |
| **Building Lookup** | ~1,800 lines of duplicated switch-case across 3 files | Single `findBuilding()` function + data array with aliases |
| **Food/Contacts** | Static HTML tables with inline styles | Searchable, filterable React components with responsive grids |
| **Styling** | Duplicated CSS across 3 files, no consistent theme | Tailwind utility classes, CSUF brand color system (navy/orange) |
| **Responsive** | Desktop only | Mobile bottom sheet, collapsible sidebar, responsive card grids |
| **Dependencies** | Vendored Leaflet source committed to repo | npm packages, .gitignore'd node_modules |
| **Security** | API token bundled in client JS | No API keys or tokens anywhere in the codebase |

## Tech Stack

- **Framework**: Next.js 16 (App Router)
- **Language**: TypeScript
- **Mapping**: react-leaflet + Leaflet.js
- **Styling**: Tailwind CSS v4
- **Events Data**: CSUF public JSON API (build-time fetch)
- **Routing**: Haversine distance + polyline (no external API)

## Getting Started

```bash
npm install
npm run fetch-events   # downloads latest CSUF events
npm run dev            # starts dev server at localhost:3000
```

`npm run build` automatically fetches fresh events before building.

## Project Structure

```
src/
  app/
    page.tsx              # Map page (default route)
    contacts/page.tsx     # Department directory
    food/page.tsx         # Restaurant directory
    api/events/route.ts   # Serves local events JSON
  components/
    map/                  # CampusMap, BuildingMarkerLayer, RoutingControl, etc.
    sidebar/              # Sidebar, BottomSheet, EventCard, etc.
    directory/            # ContactList, FoodGrid, SearchInput
    layout/               # Navigation, MobileDrawer
  data/
    buildings.ts          # 63 buildings with coordinates + aliases
    departments.ts        # 42 departments with phone/email/room
    restaurants.ts        # 10 nearby restaurants
    events.json           # Auto-generated by fetch-events script
  lib/
    building-lookup.ts    # Name/alias → building resolver
```

## No API Keys Required

Everything runs without secrets. The CSUF events API is public, map tiles come from OpenStreetMap, and walking distance is calculated client-side with the haversine formula.
